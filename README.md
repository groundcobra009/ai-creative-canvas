# AI Creative Canvas

AI Creative Canvasは、画像・テキスト・図形を配置しながら、生成AIと一緒にnote見出し画像などを制作する個人向けWebアプリです。

現在は、1280×670pxのnote見出し画像を手動編集できるキャンバスに加え、Gemini画像モデルによる画像生成・編集とSeedanceによる動画生成を利用できます。

## 現在利用できる機能

- note推奨サイズ（1280×670px）のアートボード
- テキスト、四角形、円、画像の追加
- 移動、拡大縮小、回転、複製、削除
- レイヤーの表示、ロック、前後移動
- Undo / Redoとキーボードショートカット
- SQLiteへの自動保存
- アップロード画像のローカル保存
- Gemini画像モデルによる1案・3案生成
- 選択中の画像をもとにしたAI画像編集
- 生成ジョブの進捗、履歴、キャンセル、再実行
- 記事タイトル・本文・想定読者からnote見出し画像を3案作成
- 写真中心・抽象イラスト・タイポグラフィの方向性と短縮コピー確認
- 3案の縮小プレビュー、切り替え、個別レイヤー編集
- 文字のセーフエリア表示と1280×670px・10MB以下の出力検査
- 自然言語による移動・サイズ・文字・色・表示・画像再生成
- AI操作の適用前プレビュー、会話履歴、1回のUndo
- 複数デザイン案から配色・要素を組み合わせるCanvas Agent
- テキストまたは選択画像からのSeedance動画生成
- 動画ジョブの永続進捗、失敗・タイムアウト時の再実行、MP4再生・保存
- PNG / JPEG書き出し

## セットアップ

```bash
npm install
cp .env.example .env.local
npm run dev
```

ブラウザで表示されたローカルURLを開きます。データは初期設定では `data/` 以下へ保存されます。

Gemini APIキーが未設定の場合は、UIと保存フローを確認できるローカルのモック画像生成が自動的に使われます。実際のAI画像生成を利用する場合は、`.env.local` にGemini APIキーを設定してください。

```env
AI_IMAGE_PROVIDER=auto
GEMINI_API_KEY=your_api_key
SEEDANCE_API_KEY=
AI_VIDEO_PROVIDER=auto
```

APIキーには `NEXT_PUBLIC_` を付けないでください。キーはサーバー側だけで読み込み、ブラウザへ送信しません。

`AI_IMAGE_PROVIDER` は `auto`、`gemini`、`mock` を指定できます。`auto` はAPIキーがあればGemini、なければモックを選択します。モデルIDは `.env.example` の各 `GEMINI_IMAGE_MODEL_*` で上書きできます。

Canvas Agentも同じ `GEMINI_API_KEY` をサーバー側で使用します。`AI_AGENT_PROVIDER=auto` はキーがあればGeminiの構造化出力、なければ日本語対応のローカル安全モードを選択します。テキストモデルは `AGENT_TEXT_MODEL` で変更できます。

動画生成は `AI_VIDEO_PROVIDER=auto` の場合、`SEEDANCE_API_KEY` があれば火山方舟のSeedance、なければ再生可能なローカルモックを使用します。キーは火山方舟コンソールで発行したAPI Keyを設定してください。既定モデルは `doubao-seedance-1-5-pro-251215` で、`SEEDANCE_MODEL` から変更できます。

## note見出し画像スキル

左サイドバーの「記事から3案をつくる」へ記事タイトル・本文・想定読者を入力します。分析後に、抽出された読者価値・キーワードと3案のデザイン方向・短縮コピーを確認できます。コピーと方向は生成前に修正できます。

「3案をキャンバスに生成」を押すと、写真中心・抽象イラスト・タイポグラフィの編集可能なアートボードを作成します。キャンバス上部の縮小プレビューから案を切り替え、文字・図形・画像を通常のレイヤーとして編集してください。紫色の破線は一覧表示で文字を守るためのセーフエリアで、書き出し画像には含まれません。

写真中心案は、キャンバスに保存済み画像があればそれを使用します。画像がない場合やAI画像生成に失敗した場合は編集可能なビジュアル枠を配置するため、レイアウト作業を継続できます。

note公式では画像のアップロード上限が20MBへ拡大されていますが、本アプリでは軽量な見出し画像を維持するため、より厳しい10MB以下を品質基準としています。

## Canvas Agent

左サイドバーのCanvas Agentへ「タイトルを72pxにして右へ50px」「選択要素を青にして」「案2の配色を使って」などと入力します。エージェントは直接キャンバスを変更せず、許可された変更コマンドと概要を先に表示します。内容を確認して「適用する」を押した変更は、複数コマンドでも1回のUndoでまとめて戻せます。

許可される操作は移動、サイズ、文字サイズ、文字内容、塗り・線色、表示状態、別案の配色・同種要素のコピー、保存済み画像の再生成だけです。存在しない要素ID、壊れたJSON、異なる種類の要素コピー、画像再生成と他操作の混在は拒否されます。

## Seedance動画生成

左サイドバーの「Seedanceで動画をつくる」を開きます。「画像から」では、キャンバス上の保存済み画像を選択してから、動き・カメラ・雰囲気、秒数、比率、画質、音声の有無を指定します。「テキストから」は画像を選ばずに利用できます。

生成状態はSQLiteへ保存されるため、ページを再読み込みしても履歴から追跡できます。完了したクラウド動画は期限付きURLのまま保持せず、`data/generated/videos/` へMP4として保存します。一時的な取得エラーは最大3回再試行し、12分を超えたジョブはタイムアウトとして再実行できます。個人利用での意図しない重複課金を避けるため、既定の同時実行数は1です。

## データ保存

```text
data/
├── ai-creative-canvas.db
├── uploads/
└── generated/
    ├── images/
    └── videos/
```

`data/` と `.env.local` はGit管理されません。バックアップする場合は、アプリを停止してから `data/` ディレクトリ全体をコピーしてください。
